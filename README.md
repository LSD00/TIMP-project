Документация класса AuthManager

Класс AuthManager предоставляет готовый интерфейс для управления пользователями и их авторизацией. Все операции с базой данных (PostgreSQL), создание таблиц и безопасное хэширование паролей происходят автоматически внутри класса.
Подключение

Для работы необходимо слинковать библиотеку pqxx.
При компиляции добавьте флаги: -lpqxx -lpq.

Основные методы
1. Конструктор AuthManager()

    Что делает: Устанавливает соединение с базой данных.

    Особенности: При первом запуске автоматически создает таблицу users и подключает необходимые расширения для безопасности.

    Ошибки: Если база данных недоступна (неверный адрес/пароль), выбрасывает исключение std::runtime_error.

2. bool registerUser(login, password)

    Назначение: Регистрация нового пользователя в системе.

    Аргументы: Строковый логин и пароль (в открытом виде).

    Результат: * true — пользователь успешно создан.

        false — ошибка (например, такой логин уже занят).

    Безопасность: Пароль сохраняется в БД только в виде зашифрованного хэша.

3. bool authenticate(login, password)

    Назначение: Проверка прав доступа (Авторизация).

    Аргументы: Введенные пользователем логин и пароль.

    Результат: * true — логин существует и пароль верный.

        false — логин не найден или пароль не совпадает.

4. bool deleteUser(login)

    Назначение: Полное удаление учетной записи из базы данных.

    Аргументы: Логин пользователя.

    Результат: true при успешном удалении, иначе false.
