Подробное описание

Главный класс TCP-сервера.

Сервер принимает подключения от клиентов, читает JSON-сообщения, обрабатывает открытые запросы (авторизация, регистрация) и защищенные запросы (вычисление графиков), доступные только после входа.

Главный маршрутизатор (роутер) входящих запросов.

Разбор JSON-команды клиента и вызов нужного модуля.

Анализирует поле action из JSON, проверяет права доступа (авторизован ли клиент) и направляет запрос в соответствующую подсистему (AuthManager или вычисления).

Аргументы
clientУказатель на сокет клиента, отправившего запрос.
requestРазобранный JSON-объект запроса.

Ядро серверной логики. Разделено на 2 части:

  1. Открытые эндпоинты (логин, регистрация, восстановление пароля).
  2. Защищенные эндпоинты (вычисление диаграмм). Доступны только при флаге true в authenticatedClients.

privateslot

Слот, вызываемый при поступлении новых данных от клиента.

Чтение входящего потока данных из сокета.

Читает данные, пытается распарсить их как JSON и, в случае успеха, передает объект в handleRequest().

Метод читает все доступные байты, преобразует их в QJsonDocument и, если формат корректен, передает в обработчик handleRequest.

Отправка стандартизированного ответа на запросы аутентификации.

Формирует и отправляет JSON-ответ для операций аутентификации.

Формирует JSON с полями status и message на основе кода AuthResult.

Аргументы
clientЦелевой сокет клиента.
resultРезультат операции из модуля AuthManager.
customErrorMsgПользовательский текст ошибки (если стандартный не подходит).

Если операция прошла успешно, возвращает {"status": "success"}. В противном случае возвращает {"status": "error", "message": "..."}.
